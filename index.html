프로젝트 제출 사항

  • Project Design Proposal - MS Office 파워포인트 파일(*.pptx) 혹은 PDF 파일(*.pdf)로 지정된 일시까지 제출 후 발표하여야 함(아래 내용을 개요로 하는 발표용 ppt or keynote 파일 작성)(Due : 11/20 23:59:00)
    • 개발할 서비스의 명칭 및 주된 기능 설명
    • 프로젝트 Django Project Design Outline - 프로젝트는 최소 2개의 Django Application, 최소 6개 이상의 Model을 포함하여야 합니다.
    • 프로젝트에서 포함할 세부 기능(View) 목록
    • 아래 개요를 참고하여 팀이 개발할 서비스의 내용과 구조를 소개하는 발표자료를 작성하여 제출할 것
    • e.g. 성균관대 방알리미 - 성균관대 주변 원룸 평가 시스템
      • 주요기능 : 성균관대 자연과학캠퍼스 근방의 학생들이 거주하는 원룸에 대한 평가를 기록하는 웹 서비스
      • Django Project Design Outline
        • Applications
          • "estate" - 원룸과 관련된 기본 정보 저장과 처리를 다루는 Application
          • "review" - 원룸에 대한 평가(평점, 후기)와 관련된 정보 저장과 처리를 다루는 Application
          • "user" - 서비스의 사용자(회원)과 관련된 정보 저장과 처리를 다루는 Application
        • Models
          • estate
            • Building - 건물에 대한 정보를 저장하는 Model(우편번호, 주소, 층고, 학교에서 부터의 거리 등)
            • Room - 건물에 속한 개별 원룸에 대한 정보를 저장하는 Model(호수, 사용면적, 옵션 목록 등)
            • Owner - 건물주에 대한 정보를 저장하는 Model(이름, 성격, 건물 관리 상태 등)
          • review
            • BuildingReview - 건물 자체에 대한 평가(평점, 서술형 평가, 사진 등)
            • RoomReview - 건물에 속한 개별원룸에 대한 평가를 저장하는 Model(평점, 서술형 평가, 사진, 방의 특징 등)
          • user
            • MembershipUser - 서비스 사용자 관련 정보(이메일 주소, 성대 학생 여부 등)
      • 세부기능
        • 회원 가입/로그인/로그아웃
        • 건물 정보 등록/수정
        • 방 정보 등록/수정
        • 건물주 정보 등록/수정
        • 건물 평가 등록/수정/조회
        • 방 평가 등록/수정/조회
  • 프로젝트 최종 산출물 1 - 프로젝트 소스코드 : 팀별 Github Private Repository를 만들고 각 팀원들을 구성원으로 넣은 이후 최종 제출 시점에 담당 교수도 구성원으로 추가하여야 함
  • 프로젝트 최종 산출물 2 - 실제 동작하는 웹사이트 : 완성된 프로젝트 소스코드를 무료 호스팅 서비스인 PythonAnywhere 에 배포하여 URL을 통해 실제 동작하도록 시연하여야 함

프로젝트 유의 사항

  • 프로젝트를 수강생이 직접 진행하지 않고 수강생 이외의 인원이 코드를 대리 작성하는 행위는 엄격히 금지됨.(발견 즉시 F)
  • 모든 팀원이 프로젝트 코드 개발에 참여야하여야 하며 Git Commit Log에 1회 이상 등장하지 않을 경우 프로젝트 미참여로 간주.
  • 모든 코드는 Python 3.7.x / Django 2.2.x 에서 정상동작 하도록 작성되어야함.
  • 강의 자료 이외의 Django 관련 서적, 인터넷 자료 검색 등을 적극적으로 활용할 것이 권장 됨.
  • Django Project는 PyCharm을 사용하지 않을 경우 개발에 많은 어려움이 따르므로, 반드시 PyCharm 및 Debugging 기법을 숙지 할 것
